# User Stories

**Purpose**: Convert requirements into user-centered stories with acceptance criteria

## Prerequisites
- Workspace Detection must be complete
- Requirements Analysis recommended

## Intelligent Assessment
**Execute if**: User-facing features, UX changes, multi-persona systems, customer-facing APIs, complex business logic, cross-team projects
**Skip if**: Pure refactoring, isolated bug fixes, infrastructure-only, dev tooling, documentation
**Default**: When in doubt, include user stories

## PART 1: PLANNING

### Step 1: Validate User Stories Need
Create assessment document in `aidlc-docs/inception/plans/user-stories-assessment.md`

### Step 2-4: Create Story Plan
Generate comprehensive plan with story artifacts:
- `stories.md` — User stories following INVEST criteria
- `personas.md` — User archetypes with characteristics
- Acceptance criteria for each story

### Step 5-8: Questions and Answers
Generate context-appropriate questions in `story-generation-plan.md`
Use [Answer]: tag format. Wait for ALL answers. Analyze for ambiguities.

### Step 9-14: Approval
Resolve ALL ambiguities with follow-up questions. Get explicit user approval.

## PART 2: GENERATION

### Step 15-23: Generate Artifacts
Execute approved plan. Generate stories.md and personas.md.
Update checkboxes immediately. Present completion message.
Wait for explicit user approval.

## Critical Rules
- Context-appropriate questions only
- NO proceeding with ambiguity
- Explicit approval required before generation
- Follow plan exactly, update checkboxes immediately
